# 原始需求文档 (v2)

> 保存用户原始需求，用于后续参考

## 核心需求

### 1. 整体目标
在S2T文件夹中创建统一架构，可以不用硬搬原项目代码，技术上优先参考其实现方案或思路；全方案可配合本地模型离线闭环运行；充分利用几个项目里面的模型文件（比较大，不想重新下载了）

### 2. 声音来源支持
- 原项目支持的几种音频文件
- 系统麦克风实时录入
- 内部系统声音播放（如网页声音、软件声音）

### 3. 转录功能
- 支持发言人分离
- 可配置热词

### 4. 批量并发
- 转录支持批量并发，仅文件支持，实时录入不支持
- 并发数可配置

### 5. 模型配置
- 相关模型都可以通过ollama或本地文件夹进行模型配置
- 支持对接阿里百炼、Deepseek的在线大模型API（需配置key）
- 只配置一个当前使用的API

### 6. 项目管理
- 支持项目管理
- 支持一个项目多个文件
- 转录后内容有单独页面展示
- 页面上可以对文字进行修改

## UI重构需求 (v2)

### 1. 取消会话层级
- 项目直接包含音频文件，不需要会话层级

### 2. 项目详情页布局
**上部分 - 项目信息卡：**
- 左右宽度自适应
- 包含：项目创建时间、描述、热词列表
- 热词以英文逗号间隔显示和编辑
- 热词列表尾部有编辑按钮，点击弹窗修改

**下部分左侧 - 内容列表：**
- 【添加】按钮：创建音频，选择来源类型（上传文件/外部录音/系统内录音）
- 【AI问答】按钮：进入AI问答状态
- 列表显示：音频名称、类型、创建时间、转写状态、操作栏
- 文件上传：待转换、转写中、已完成
- 实时录音：转写中、已完成（无进度）
- 转写中可"停止"，已完成可"播放"

**下部分右侧 - 多功能详情页：**
- 默认状态：项目已转写内容列表（卡片收缩/展开）
- 对照回顾状态：音频播放+AI总结+字幕同步
- AI问答状态：问答界面+参数设置+流式输出

### 3. 实时录音
- 实时转写并同时保存录音文件到服务器
- 如果已在录音，其他录音类型不可选择

### 4. AI问答
- 用户提示词输入
- 项目所有转写内容作为系统提示词
- 参数：上下文轮次(默认3)、温度(默认0.4)
- 流式输出

### 5. UI风格
- 苹果现代风格设计
- 引用uiuxpromax skill能力
